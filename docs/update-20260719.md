## Bugs

### Bug 1: Walk-In Discount Display on /pos History ⚡ S | Frontend-only

**Problem:** Kanban cards correctly show strikethrough gross + net when discounted, but History modal only shows net amount with no discount indication.

**Root cause:** `frontend/js/pos-history.js` line ~43 only renders `RM ${o.totalAmount.toFixed(2)}`. No discount logic exists in history cards.

**Fix:** In `pos-history.js` `itemHtml()`, replicate the Kanban pattern:
- Compute `gross = totalAmount + discountOffset`
- If `discountType !== 'NONE'` and `discountOffset > 0`, show `<s>RM gross</s> → RM net`

**Files:** `frontend/js/pos-history.js`

### Bug 3 + Bug 4: Reports.html Login / Persistent Login ⚡ S | Frontend-only

**Problem:** Navigating from admin to reports.html requires re-login. Also, closing tab and revisiting reports requires login again.

**Root cause:** `admin.js` stores token in `localStorage`, but `reports.js` reads/writes `sessionStorage`. sessionStorage doesn't persist across tabs or sessions.

**Fix:** In `frontend/js/reports.js`, replace all `sessionStorage` references with `localStorage` (lines ~16–17 reads, ~92–93 writes, logout removes). This fixes both bugs with one change.

**Files:** `frontend/js/reports.js` (~6 line replacements)

---

### Bug 5: Activity Log "Opened"/"Closed" Shows "—" ⚡ S | Backend-only

**Problem:** Report > Summary > Activity Log always shows "—" for Opened/Closed columns.

**Root cause:** `backend/src/routes/checklist.ts` line ~68 computes `allCompleted` flag by checking ALL config items including disabled ones. If any disabled item exists in config but isn't checked, `allCompleted` stays false, and the frontend shows "—".

**Fix:** Filter to only enabled items before the `every()` check:
```ts
// Before: phaseItems.every(i => items[i.id]?.checked)
// After: phaseItems.filter(i => i.enabled !== false).every(i => items[i.id]?.checked)
```

**Files:** `backend/src/routes/checklist.ts`

```js

  ? `[PRE-ORDER: ${preorderRecord.code}] Collect: ${collectionTime}${trimmedNotes ? ' | ' + trimmedNotes : ''}`
  : trimmedNotes;
```
